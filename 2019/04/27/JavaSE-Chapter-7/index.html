​ 异常处理。

异常处理

​ Java的程序执行过程中发生的异常大致分为两类.

​ Error: Java 虚拟机无法解决的严重问题，一般不编写针对性代码处理。

​ Exception: 编程错误或偶然的外因导致的一般问题，可以使用针对性代码进行处理。

异常

程序在运行中出现的非正常状况

​ 程序出现异常如不进行处理，程序可能会崩溃。

分类

  • 按程度区分
    1. Error: JVM也无法进行处理的严重错误
    2. Exception: 一般异常
  • 按照处理方式区分
    • 受检异常(checked)：编译时异常，在程序中必须处理
    • 非受检异常(unchecked): 运行时异常，在程序中无法处理

ps. RuntimeException属于Error

异常机制

​ 异常是一个对象，产生异常将会逐级向上层调用者抛出，直至异常完成处理

异常处理

捕获

​ 使用使用try-catch-finally语句捕获处理异常

​ 可使用组合

  • try-catch
  • try-catch-finally
  • try-finally

抛出

​ throw语句可用来从方法中主动抛出指定异常对象。可以主动抛出RuntimeException及其子类的异常对象。

​ 在方法签名中使用throws指明本方法可能抛出的异常，提醒调用者。

​ throw语句执行后，方法返回携带抛出的异常对象。

​ 一些情况下会先捕获异常，然后将该异常包装为自定义异常后抛给调用者。

异常抛出

​ 入口方法或方法出现问题将引起严重的后果时使用捕获。

​ 非入口，被调用的方法，应抛出异常。
